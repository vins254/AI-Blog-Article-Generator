{# ============================================================================= FILE: client/templates/login.html PURPOSE: The user login / authentication page. This is the gateway to the application — unauthenticated users are redirected here automatically by Django's @login_required decorator on all protected views. ============================================================================= HOW IT WORKS: - This template renders a simple HTML form with two fields: username & password. - When the form is submitted (POST), Django's built-in authenticate() function checks the credentials against the hashed passwords stored in the database. - If authentication succeeds, the user is logged in via login() and redirected to the main dashboard (/). - If it fails, the view sends back an 'error_message' variable which this template conditionally displays using the {% if error_message %} block. - If the user is already logged in, the view skips all of this and redirects them straight to the dashboard so they don't see the login page unnecessarily. FEATURES & TOOLS: - Django Template Language (DTL): {% if error_message %} for conditional rendering of server-side error feedback without any JavaScript. - {% csrf_token %}: Injects a hidden security token into the form. Django will reject any POST request that doesn't include this token, protecting against Cross-Site Request Forgery (CSRF) attacks. - autocomplete attributes: Set to 'username' and 'current-password' to tell the browser's password manager how to auto-fill fields correctly. - {% load static %}: Allows us to load the shared styles.css from the client/static/css/ directory using Django's static file resolver. ============================================================================= #} {% load static %} Login — ContentFlow
